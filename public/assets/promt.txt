TABEL: siswa

STRUKTUR DATABASE:
- id (INT, PK, AUTO_INCREMENT)
- nis (VARCHAR(12), UNIQUE)
- nama_siswa (VARCHAR(255))
- id_kelas (INT, FK ke kelas.id_kelas)
- angkatan (INT)
- status (ENUM: aktif, lulus, pindah, dropout)
- created_at, updated_at

MODEL: Siswa (sudah dibuat / belum dibuat)

RELASI:
- belongsTo Kelas (via id_kelas)

FITUR CRUD:
CREATE:
- Form: nis, nama_siswa, pilih kelas (dropdown), angkatan, status (dropdown)
- Validasi: NIS harus unik, 10-12 digit

READ:
- Tabel kolom: No, NIS, Nama, Kelas, Angkatan, Status, Aksi
- Filter: berdasarkan kelas, status, angkatan
- Search: nama dan NIS
- Pagination: 25 per halaman

UPDATE:
- Bisa edit semua field kecuali NIS (read-only)

DELETE:
- Soft delete (data tidak benar-benar dihapus, cuma status jadi deleted_at)

FITUR TAMBAHAN:
- Import dari Excel
- Export ke Excel dan PDF
- Bulk delete
- Badge warna: aktif (hijau), lulus (biru), pindah (kuning), dropout (merah)

Kalo mau set 'true' record lain disaat sudah ada yg active/true, maka akan muncul card/pop-up notification untuk konfirmasi apakah yakin mau mengubah tahun yg aktif, jika user setuju, tahun aktif yg lama diubah statusnya jadi false, sedangkan tahun yg baru diubah statusnya jadi true.
 - Filter: berdasarkan year_name dan status
 - Search: year_name
 - Pagination: 10 per halaman

UPDATE:
 - Bisa edit semua field, ganti year_name dan is_active. Untuk is_active sama yg udh aku jelasin di bgian Read:Tabel Kolom, Kalo mau set 'true' record lain disaat sudah ada yg active/true, maka akan muncul card/pop-up notification untuk konfirmasi apakah yakin mau mengubah tahun yg aktif, jika user setuju, tahun aktif yg lama diubah statusnya jadi false, sedangkan tahun yg baru diubah statusnya jadi true.

Delete:
 - Delete ya, langsung ilang.
 - Sebelum menghapus harus ada validasi, 2 KALI, karena kalo data ini dihaps bakal ngaruh ke tabel lain yg kemungkinan data di tabel lain bakal terhapus.
 - Validasi 1: Cuman konfirmasi iya/tidak, dengan teks, "Apakah anda yakin ingin menghapus data tersebut?" Kalo bisa teksnya sesuain lagi, sama ditambah teks peringatan apa yg bakal terjadi kalo data ini dihapus.
 - Validasi 2: Agar bisa melanjutkan penghapusan data penting ini, user harus memasukan email dan password lagi, kalo sesuai, ini bakal langsung terhapus, biar gk ada orang iseng yg nyoba2 ngehapus data ini
 - Jika memungkinkan, validasi kedua cuman akan muncul jika memang ada data dari tabel lain yg terhubung dengan tahun yg dipilih. Tapi kalo tahun yg dipilih emang gk dipake, Validasi 2 gk bakal muncul. Ini biar kalo admin gk sengaja bikin tahun yg salah, bisa langsung dihapus, gk perlu repot  validasi 2 kalo, toh datanya juga kosong.

Fitur tambahan:
 -  Untuk Bulk Delete, sistem hapusnya sama kayak Delete biasa, harus ada validasi 2 kali.
 - Badge warna: Hijau dengan simbol centang jika status true, jika false Merah dengan simbol silang
